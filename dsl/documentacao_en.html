introduction

what is this project?

automated pipeline for cfd simulation of packed beds using a custom dsl, blender for 3d geometry, and openfoam for the flow solve.

custom dsl

.bed language to describe bed parameters in a concise way

3d geometry

automatic generation of realistic 3d models in blender with physics

cfd simulation

integration with openfoam for flow analysis

containerized

docker compose for reproducible environments

system architecture

user
defines parameters via terminal setup or a .bed file
↓
antlr compiler
validates syntax and produces file.bed.json
↓
blender (headless)
builds 3d geometry with particle physics
↓
openfoam
cfd simulation of the flow
↓
visualization
web dashboard with results

installation

prerequisites

  • python 3.8+ - main language
  • java 11+ - to run antlr
  • blender 3.0+ - geometry generation
  • git - version control

quick install

1. clone the repository

git clone https://github.com/bengo501/CFD-PIPELINE-TCC-1.git
cd CFD-PIPELINE-TCC-1

2. install python dependencies

pip install antlr4-python3-runtime

3. generate the antlr parser

cd dsl
java -jar antlr-4.13.1-complete.jar -Dlanguage=Python3 -o generated grammar/Bed.g4

done! you can launch the terminal setup:

python bed_wizard.py

quick start

mode 1: interactive setup

python bed_wizard.py
# start > 3d generation (or full questionnaire)
# fill in parameters
# wait for generation

mode 2: edit a template

python bed_wizard.py
# choose the templates path from the menu
# edit the .bed file
# compile and run

mode 3: create a file by hand

// file: my_bed.bed
bed {
    diameter = 0.05 m;
    height = 0.1 m;
    wall_thickness = 0.002 m;
}

particles {
    kind = "sphere";
    count = 100;
    diameter = 0.005 m;
    density = 2500.0 kg/m3;
}

packing {
    method = "rigid_body";
    gravity = -9.81 m/s2;
}

then compile:

python compiler/bed_compiler_antlr_standalone.py my_bed.bed

main parameters

bed section (geometry)

parameter range description
diameter 0.01 - 2.0 m inner bed diameter
height 0.01 - 5.0 m bed height
wall_thickness 0.0001 - 0.1 m wall thickness

particles section

parameter range description
kind sphere, cube, cylinder particle shape
count 1 - 10000 number of particles
diameter 0.0001 - 0.5 m particle diameter
density 100 - 20000 kg/m³ material density

packing section

parameter range description
method rigid_body packing method
gravity -50 to 50 m/s² gravitational acceleration
substeps 1 - 100 simulation substeps / accuracy

tip: use the built-in help

in the terminal setup, type ? on a field for details, allowed ranges, and examples.

typical workflow

1. define parameters

  • open setup: python bed_wizard.py
  • choose start > 3d generation or full questionnaire (engine first in json)
  • fill fields using contextual help

2. generate geometry

  • setup compiles the .bed automatically
  • runs blender headless
  • saves the model under generated/3d/output/

3. view the result

three main modes: browser (three.js in the frontend, list via fastapi), desktop (open3d script for stl/obj/ply), and blender (native .blend scene; cad meshes imported via scripts/blender_scripts/open_imported_mesh_gui.py). more detail: docs/visualizacao_3d_arquitetura.md.

  • in setup, 3d visualization lists files (e.g. local_data/models_3d/)
  • web: url with ?meshViewerId=… — requires npm run dev and a running api
  • help menu, option 7: summary of modes and the format compatibility table
